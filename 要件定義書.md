## 1. プロジェクト概要

資産形成・FIRE・投資テーマのYouTube動画を自動生成・投稿し、再生データを自律的に確認してPDCAを回し続けるプログラム。人間の操作を必要としない完全自動システム。

### 中心概念
1. **①生成**：動画のスクリプト・サムネ・アップロードまでを自動で行う
2. **②仮説**：どのターゲットに、どんなニーズを満たしに行くか仮説を立てて言語化した上で動画生成
3. **③計測**：動画の再生回数・実績データを自律的に確認
4. **④PIVOT**：再生数が悪ければコンテンツと仮説をPIVOTして次の動画を自動生成
5. **⑤継続**：YouTubeの無料API上限数までPDCAを回し続ける（人間の操作不要）

## 3. 1日の自動フロー

macOS launchd により毎日09:00に自動実行。

```
[Step 1] 48h経過動画のパフォーマンスチェック
  → state.jsonから「投稿から48h以上経過・未チェック」の動画を抽出
  → YouTube API で視聴回数を取得
  → 30回以上 → status: "success"、成功パターンを記録
  → 30回未満 → status: "pivoted"、PIVOTフラグを立てる

[Step 2] 新規動画を2本生成・投稿（繰り返し×2）
  → 仮説生成（テーマ × ターゲット × ニーズ）
  → スクリプト生成（8スライド分の台本）
  → スライド画像生成（Pillow、1920x1080）
  → TTS音声生成（macOS say -v Kyoko → AIFF → WAV）
  → FFmpegで動画結合（MP4出力）
  → YouTubeにアップロード
  → state.jsonに動画ID・仮説・投稿時刻を記録

[Step 3] ログ出力して終了（次の実行まで待機）
```

## 6. YouTube API 仕様

### クォータ消費
| 操作 | 消費ユニット |
|---|---|
| 動画アップロード（1本） | 1,600ユニット |
| 視聴統計取得（videos.list） | 1ユニット |
| 1日の無料上限 | 10,000ユニット |
| 1日2本運用時の消費 | 約3,210ユニット（32%） |

### 必要スコープ（既存から変更）
| 既存 | 変更後 |
|---|---|
| `youtube.upload` | `youtube` （フルアクセス） |

> 初回起動時にブラウザ再認証が1回必要。以降はtoken.jsonで自動。

### 費用
| 項目 | 費用 |
|---|---|
| YouTube Data API v3 | **無料**（10,000ユニット/日） |
| macOS TTS（sayコマンド） | **無料** |
| FFmpeg | **無料** |
| Pillow（画像生成） | **無料** |
| **合計** | **$0** |

```json
{
  "videos": [

      "video_id": "abc123xyz",

      },
      "title": "新NISAの始め方【30代会社員が知るべき3つのポイント】",
      "views_48h": null,
      "status": "pending_check"
    }
  ],
  "used_themes": ["nisa_basics"],
  "pivot_count": 0,
  "success_patterns": [],
  "last_run": "2026-03-06T09:00:00"
}
```

### statusの状態遷移
```
pending_check → (48h経過後チェック) → success（≥30回）
                                    → pivoted（<30回）
```
